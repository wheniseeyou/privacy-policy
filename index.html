This Privacy Policy describes in detail how HealthCare+ ("we", "us", "the App") collects, uses, stores, protects, and shares your personal and health-related information. By using the App, you agree to these practices.

1. Privacy-First Architecture

HealthCare+ is designed with a privacy-first architecture. The vast majority of your data is stored locally on your device using Apple SwiftData with iOS system-level encryption. We do not operate backend servers for user data storage.

2. HealthKit Data - Detailed Disclosure

The App reads and writes health data via Apple HealthKit. Access requires your explicit permission. Revoke anytime: iOS Settings > Privacy and Security > Health > HealthCare+.

  • Mindful minutes - record meditation sessions
  • Water consumption - record hydration events
  • Caffeine intake - record coffee consumption

2.3 How Health Data Is Processed

  • Local only: All analysis (stress score, recovery score, focus score) runs on-device. No health data is transmitted to any server.
  • HRV stress calculation: Reads HRV samples, computes stress percentage based on your personal 7-day baseline. Runs locally.
  • Background delivery: Registers HKObserverQuery for HRV, heart rate, sleep, and step updates to keep widgets/Live Activities current. Data processed locally.
  • No advertising use: Health data is NEVER used for advertising, marketing, profiling, or sold to third parties.
  • No data mining: We do not aggregate, anonymize, or mine health data for analytics.

Always-on location: Only for Golden Buddy landmark detection using iOS region monitoring (battery-efficient). Raw GPS coordinates of landmark visits are NOT stored. Revoke: iOS Settings > Privacy > Location Services > HealthCare+.

4. Audio / Microphone (Snore Detection)

  • Framework: Apple SoundAnalysis (on-device ML classifier)
  • Function: When user manually activates snore detection (before sleep), microphone captures audio, SoundAnalysis classifies snoring locally.
  • Audio storage: Short clips (seconds) of detected snoring stored locally for playback. NO audio transmitted to servers.
  • Live Activity: Results shown via Dynamic Island during monitoring.
  • User control: Never auto-activated. User must manually start each time. Revoke: iOS Settings > Privacy > Microphone > HealthCare+.

5. CloudKit / iCloud Sync

Optional cross-device sync via Apple CloudKit (public database):

  • Synced: User profile (anonymized), buddy collection, badges, coffee/discipline/mood/food records, location events (landmark name only)
  • Container: iCloud.com.yaotianshu.tinyFolish
  • Encryption: Apple CloudKit encryption (server-side + TLS in transit)
  • Developer access: We CANNOT access your synced data content - Apple's CloudKit privacy model prevents this.
  • HealthKit data NOT synced via CloudKit. Health data stays on each device independently.

6. Sign in with Apple

  • Received: Anonymous User Record ID (e.g., "_abc123def456"). NOT your Apple ID email or name.
  • NOT received: Your Apple ID email, real name, phone number, or any PII from Apple.
  • Nickname and avatar: Chosen by you within the App. Not imported from Apple ID.
  • Account deletion: Settings > Account > Delete Account. Clears CloudKit + SwiftData + UserDefaults + widget data.

13. Your Rights

  • Withdraw consent: Revoke HealthKit/Location/Microphone in iOS Settings anytime

14. Security Measures

  • SwiftData - iOS sandbox + file encryption
  • CloudKit - Apple-managed encryption (server-side + TLS)
  • API communication - HTTPS/TLS
  • No data brokerage - we do not sell, rent, or trade user data
