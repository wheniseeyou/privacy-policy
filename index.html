HealthCare+ is designed with a privacy-first architecture. The vast majority of your data is stored locally on your device using Apple SwiftData with iOS system-level encryption. We do not operate backend servers for user data storage.

2. HealthKit Data - Detailed Disclosure

The App reads and writes health data via Apple HealthKit. Access requires your explicit permission. Revoke anytime: iOS Settings > Privacy and Security > Health > HealthCare+.

3. Location Data (CoreLocation)

Workout route GPS (when-in-use), Golden Buddy landmark detection (1km region monitoring, stores landmark name + timestamp only, NOT raw coordinates), Weather lookup (city-level approximate). Always-on location only for landmark detection via iOS region monitoring (battery-efficient). Revoke: iOS Settings > Privacy > Location Services.

Use CasePrecisionPurposeStored

4. Audio / Microphone (Snore Detection)

Apple SoundAnalysis (on-device ML). When user manually activates (before sleep), microphone captures audio, SoundAnalysis classifies snoring locally. Short clips stored locally for playback. NO audio transmitted to servers. Never auto-activated. Revoke: iOS Settings > Privacy > Microphone.

5. CloudKit / iCloud Sync

Optional cross-device sync via Apple CloudKit (public database, container: iCloud.com.yaotianshu.tinyFolish). Synced: user profile (anonymized), buddy collection, badges, coffee/discipline/mood/food records, location events (landmark name only). Apple CloudKit encryption (server-side + TLS). Developer CANNOT access synced data content. HealthKit data NOT synced via CloudKit.

6. Sign in with Apple

Received: anonymous User Record ID (e.g. _abc123def456). NOT received: your Apple ID email, real name, phone number. Nickname and avatar chosen by you within App. Account deletion: Settings > Account > Delete Account clears all data.

7. AI Services (Third-Party APIs)

MiniMax M2.7 API (api.minimaxi.com): sends your typed health questions + aggregated summaries (e.g. walked 8000 steps). Does NOT send raw HealthKit samples, GPS, audio, or personal identity. Requires explicit opt-in consent. Disable: Settings > AI Analysis > off. Chat history stored locally only. Open-meteo API: sends approximate lat/long for weather. No personal data sent.

8. Photos and Camera

Avatar: PhotosUI picks ONE photo for profile. Stored locally + CloudKit (encrypted). Does NOT access full photo library. Camera: NOT used.

9. Calendar and Reminders (EventKit)

Optionally writes discipline activity reminders to iOS Calendar/Reminders. One-way write only. Does not read existing calendar data.

10. Widget Data (App Group)

Shares data with own widgets via App Group (group.com.yaotianshu.healthcare): habit progress, sleep stages, HRV, stress levels, snore status. Stays within device sandbox. Never transmitted externally.

隐私优先架构

本App采用隐私优先架构。绝大部分数据本地存储在您的设备上（Apple SwiftData + iOS系统级加密）。不运营后端服务器存储用户数据。

本App通过 HealthKit 读写健康数据。需明确授权。随时撤销：iOS设置→隐私与安全性→健康→HealthCare+。

运动路线GPS(使用时)；金色伙伴地标检测(1km区域监测，仅存地标名称+时间戳，不存原始坐标)；天气查询(城市级近似)。始终位置仅用于地标检测(iOS区域监测，省电)。撤销：iOS设置→隐私→定位服务。
